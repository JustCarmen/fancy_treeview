Description

back to top
With this module it is possible to add multiple pages for several branches in your family tree. The branches will be listed from the first generation down to the last one in a narrative way. It gives your users a quick overview of your family tree. Of course privacy rules are respected. You can view an example of a Fancy Tree View page here.

Requirements

back to top
This module requires webtrees version 1.5.0 or later.

Installing the module

back to top
Installation is very simple. Upload the folder 'fancy_treeview' to your server's webtrees /modules_v3 folder. Then login to webtrees as administrator and go to Module Administration (Admin -> Module Administration). Look down the list of installed modules for "Fancy Tree View" and ensure it is ticked as "enabled".

IMPORTANT: In order to retrieve the configuration options you must click again on the module link in the menu. This will refresh the menu and add the Fancy Tree View configuration page to the admin menu.

After the module is installed go to the module configuration page to set some options.

Updating from a previous version

back to top

This module will be updated whenever there are some bugs fixed or feature requests implemented. Each update will be announced in the webtrees forum. In case of updating you can just overwrite your previous version of the Fancy Tree View module with the new one by uploading the folder 'fancy_treeview' to your server's webtrees /modules_v3 folder. All settings will be remained after an update. It is always a good behaviour to clear the cache after each update. Go to your webtrees administration - clean up data folder and tick the box 'cache'.

Whenever you run into any problems after an update try to reset the database settings by clicking on the reset button on the Fancy Tree View configuration page. Most of the time this will solve any issues you could have after updating.

Module options

back to top
On the Fancy Tree View configuration page you have some options. To start using the module enter a surname in the textfield. The script will automatically search for the oldest person with this surname. After the search is complete, it is possible to change the ID of the root person and set several other options. You can enter as much surnames as you like. There will be a menu-link and page created for each surname. By dragging and dropping the table entries which will appear after you have set the first surname, you can sort the pages in any way you want.

Themes

back to top

Stylesheets for all the core webtrees themes and the JustBlack theme are added to the module package. So the pages will look good in each of these themes.

Custom translations

back to top

Where possible the Fancy Tree View module uses existing webtrees language items. These will therefore translate automatically wherever they are used. We have included new language items in the same "WT_I18N::translate()" format. This means you can translate these into any other language using webtrees "custom translation" feature, described here: Wiki: Custom translation. In stead of adding your custom translation file(s) to the folder data/language, you may opt to keep them seperate from other translated files by adding them to the folder modules_v3/fancy_treeview/language. Because we use complex translations in this module csv or php files as decribed on the Wiki could not be used.

You can use the default.pot file added in the module language folder, to start translate the module in other language(s). The default.pot file contains all strings needed to be translated. You can open this file in any texteditor or use it as the update file in a programm like Poedit. Save your file as *.po file. To use the translated *.po file on the module you need to convert this file to a *.mo file. This can be done with Poedit. But it is not mandatory. Just send me the translated *.po file and I will turn it in to a *.mo file and add it to the module package.

At the moment the Fancy Tree View module is already translated in nine different languages. You can find all the supported languages in the language folder. If you have any additions or improvements to these files, please send me an edited copy.
